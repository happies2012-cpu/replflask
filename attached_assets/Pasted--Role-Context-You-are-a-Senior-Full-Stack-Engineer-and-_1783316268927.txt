## TASK 1: AUTOMATED BRAND ASSET PIPELINE (CROSS-PLATFORM)

Download the master brand logo from this URL:
https://www.gsgroups.net/gslogo.png

Write a completely automated Node.js automation script (`scripts/generate-brand-assets.js`) using the `sharp` library. The script must programmatically fetch this image, handle transparency masking where required, generate the exact multi-resolution binaries, and write them directly to the specified file trees.

### 1. Web Platform & PWA Assets
- Target Directory: `frontend/public/` or `public/`
- Generate and write:
  * `favicon.ico` (Multi-resolution: 16x16, 32x32, and 48x48 embedded in a single ICO file)
  * `favicon-16x16.png` & `favicon-32x32.png`
  * `apple-touch-icon.png` (180x180, unmasked, flat solid background layer added)
  * `android-chrome-192x192.png` & `android-chrome-512x512.png`
- Action: Update `index.html` or your root layout/meta definition files to explicitly link these generated premium brand assets.

### 2. Electron Desktop Platform
- Target Directory: `electron/assets/` or `desktop/build/`
- Generate and write:
  * `icon.icns` (Apple Desktop Format - packing layers up to 1024x1024)
  * `icon.ico` (Windows Desktop Format - packing layers up to 256x256)
  * `icon.png` (Linux Desktop Format - 512x512)

### 3. Mobile Platforms (iOS & Android Mipmaps)
- Target Directories: `mobile/ios/` and `mobile/android/` (or your active Capacitor/React Native resource folders)
- iOS (`AppIcon.appiconset`): Generate clean, standard crisp sizes required by Xcode: 20x20, 29x29, 40x40, 58x58, 60x60, 76x76, 80x80, 87x87, 120x120, 152x152, 167x167, 180x180, and 1024x1024. Ensure zero alpha channel/transparency for final iOS App Store build compliance.
- Android (`mipmap-*` paths): Generate legacy launcher icons (`ic_launcher.png`) and adaptive foreground split layers (`ic_launcher_foreground.png` overlaid over a clean vector/solid background) across all target densities: mdpi (48x48), hdpi (72x72), xhdpi (96x96), xxhdpi (144x144), xxxhdpi (192x192).

### 4. Optimized In-App Brand UI Assets
- Target Directory: Your core UI components assets directory.
- Action: Optimize the master PNG into a highly compressed, web-ready `logo.png` optimized for high-density displays, and use it inside the Workspace Navigation Navbar and Login Screen layout shells.
